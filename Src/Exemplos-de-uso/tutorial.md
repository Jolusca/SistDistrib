# OpenCV tutorial

## Key Parameters Summary

### Functions for Shape Identification

- **`cv2.GaussianBlur(image, kernel_size, standard_deviation)`**
  - **Description:** Blurs the image to reduce noise.
  - **`kernel_size`:** Size of the blurring matrix (must be odd, e.g., `(5, 5)`, `(9, 9)`). Larger sizes result in smoother images.
  - **`standard_deviation`:** Usually set to `0` for automatic calculation.

- **`cv2.HoughCircles(image, method, dp, minDist, param1, param2, minRadius, maxRadius)`**
  - **Description:** Detects circles using the Hough Transform.
  - **`method`:** Usually `cv2.HOUGH_GRADIENT`.
  - **`dp`:** Inverse ratio of the accumulator resolution. `1` = same resolution as the image.
  - **`minDist`:** Minimum distance between the centers of detected circles (e.g., `50`).
  - **`param1`:** Higher threshold for edge detection (e.g., `50`).
  - **`param2`:** Threshold for circle detection (lower = more circles detected, e.g., `30` to `100`).
  - **`minRadius` / `maxRadius`:** Limits for the radius of detected circles (e.g., `10` to `1000`).

- **`cv2.adaptiveThreshold(image, max_value, method, type, blockSize, C)`**
  - **Description:** Binarizes the image using an adaptive threshold.
  - **`method`:** Usually `cv2.ADAPTIVE_THRESH_GAUSSIAN_C`.
  - **`type`:** Usually `cv2.THRESH_BINARY_INV` (inverts the values).
  - **`blockSize`:** Size of the neighborhood area used to calculate the threshold (odd, e.g., `11`).
  - **`C`:** Constant subtracted from the mean (e.g., `2` to `5`).

- **`cv2.findContours(image, mode, method)`**
  - **Description:** Finds contours in the image.
  - **`mode`:** Usually `cv2.RETR_EXTERNAL` (retrieves only the external contours).
  - **`method`:** Usually `cv2.CHAIN_APPROX_SIMPLE` (compresses the contours).

- **`cv2.contourArea(contour)`**
  - **Description:** Calculates the contour area in pixels. Used to filter out small contours (noise).
  - **Typical value:** `500` to `2000` (adjust according to your needs).

- **`cv2.approxPolyDP(contour, epsilon, closed)`**
  - **Description:** Simplifies contours into polygons.
  - **`epsilon`:** Approximation accuracy (e.g., `0.02` to `0.05` of the perimeter). The larger the value, the more simplified the contour becomes.
  - **`closed`:** Usually `True` to close the polygon.

- **`cv2.arcLength(contour, closed)`**
  - **Description:** Calculates the perimeter of the contour, used to define `epsilon` in `approxPolyDP`.

- **`cv2.boundingRect(contour)`**
  - **Description:** Returns the bounding rectangle of a contour, used to check proportions (e.g., square vs. rectangle).

### Specific Functions for Orange Color Detection

- **`cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)`**
  - **Description:** Converts the image from **BGR** color space (OpenCV default) to **HSV** (Hue, Saturation, Value).
  - **Why use HSV?**
    - The **HSV** color space is more intuitive for color detection as it separates the **Hue** from intensity and saturation.
    - It facilitates defining color ranges (e.g., orange) based primarily on **Hue**, regardless of lighting conditions.



- **`cv2.inRange(hsv, lower_orange, upper_orange)`**
  - **Description:** Creates a **binary mask** that isolates pixels within a specific color range in the HSV space.
  - **Parameters:**
    - `hsv`: Image in the HSV color space.
    - `lower_orange`: Lower limit of the orange color (`[H, S, V]`).
    - `upper_orange`: Upper limit of the orange color (`[H, S, V]`).
  - **Result:**
    - Returns a binary image (black and white) where:
      - **White**: Pixels within the orange color range.
      - **Black**: Pixels outside the orange color range.
  - **Example values for orange:**
    ```python
    lower_orange = np.array([5, 100, 100])   # H=5, S=100, V=100
    upper_orange = np.array([22, 255, 255]) # H=22, S=255, V=255
    ```

- **`cv2.findContours(mask, cv2.RETR_TREE, cv2.CHAIN_APPROX_SIMPLE)[-2]`**
  - **Description:** Finds **contours** in a binary image (mask).
  - **Parameters:**
    - `mask`: Binary image generated by `cv2.inRange`.
    - `cv2.RETR_TREE`: Retrieval mode that creates a **complete hierarchy** of contours (includes internal and external contours).
    - `cv2.CHAIN_APPROX_SIMPLE`: Compresses horizontal, vertical, and diagonal segments, saving memory.
  - **`[-2]`:**
    - Ensures compatibility between OpenCV versions.
    - In older versions, `cv2.findContours` returns 2 values: `(contours, hierarchy)`.
    - In newer versions, it returns 3 values: `(image, contours, hierarchy)`.
    - `[-2]` always extracts the **contours**, regardless of the version.

- **`cv2.contourArea(contour)`**
  - **Description:** Calculates the **area** of a contour (in pixels).
  - **Code usage:**
    - Filters out very small contours (noise) using a threshold (e.g., `area > 1000`).
    - Adjust the threshold value based on the object size and camera distance.

- **`cv2.boundingRect(contour)`**
  - **Description:** Returns the coordinates of the **bounding rectangle** of a contour.
  - **Result:**
    - Returns a tuple `(x, y, width, height)`, where:
      - `(x, y)`: Coordinates of the top-left corner of the rectangle.
      - `width` and `height`: Dimensions of the rectangle.
  - **Code usage:**
    - Used to draw a rectangle around the detected orange object.

- **`cv2.rectangle(frame, (x, y), (x + w, y + h), (0, 165, 255), 2)`**
  - **Description:** Draws a **rectangle** on the image.
  - **Parameters:**
    - `frame`: Image where the rectangle will be drawn.
    - `(x, y)`: Top-left corner of the rectangle.
    - `(x + w, y + h)`: Bottom-right corner of the rectangle.
    - `(0, 165, 255)`: Rectangle color in **BGR** (blue=0, green=165, red=255 → orange).
    - `2`: Thickness of the rectangle line (in pixels).

- **`cv2.putText(frame, "Orange Card", (x, y - 10), cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0, 165, 255), 2)`**
  - **Description:** Adds **text** to the image.
  - **Parameters:**
    - `frame`: Image where the text will be drawn.
    - `"Orange Card"`: Text to be displayed.
    - `(x, y - 10)`: Text position (10 pixels above the rectangle).
    - `cv2.FONT_HERSHEY_SIMPLEX`: Font type.
    - `0.6`: Text scale.
    - `(0, 165, 255)`: Text color in **BGR** (orange).
    - `2`: Text thickness.
